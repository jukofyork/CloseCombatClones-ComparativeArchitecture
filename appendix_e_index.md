## Game Implementation Index

### OpenCombat-SDL (2005-2008)

**Architecture**: Classical OOP with deep inheritance
**State System**: 64-bit bitfield
**Content**: XML data files
**Key Files**: chapter_02_architectural_philosophy.md (2.2.1), chapter_03_state_management.md (3.3)

### OpenCombat (2020-2024)

**Architecture**: Modified ECS with message passing
**State System**: Three-tier hierarchy (Phase/Behavior/Gesture)
**Content**: JSON deployments, TMX maps
**Key Files**: chapter_02_architectural_philosophy.md (2.2.2), chapter_03_state_management.md (3.4)

### CloseCombatFree (2011-2012)

**Architecture**: C++/QML declarative composition
**State System**: Dual-state (Runtime + Health)
**Content**: QML files (full moddability)
**Key Files**: chapter_02_architectural_philosophy.md (2.2.3), chapter_03_state_management.md (3.5)
